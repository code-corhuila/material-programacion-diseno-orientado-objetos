Semana 2: Clases y objetos en Java

En esta semana aprenderás a pensar “en objetos”: entenderás con claridad la diferencia entre clase y objeto, cómo se crean instancias en Java y cómo organizar un proyecto con varias clases. El objetivo es que puedas modelar entidades simples sin entrar todavía en detalles profundos de atributos, métodos y encapsulación (eso se abordará en la semana 3).

1. Conceptos base: ¿qué es una clase y qué es un objeto?

En Programación Orientada a Objetos (POO), una clase define una “plantilla” y un objeto es una “instancia” creada a partir de esa plantilla. Una analogía muy útil:

Clase = Plano / molde / receta (define cómo será algo).
Objeto = Instancia real creada con ese molde (algo que existe en memoria con identidad propia).
Instanciar = Crear un objeto a partir de una clase (usando new).

1.1. ¿Por qué “clase” y “objeto” son el centro de la POO?

Porque permiten modelar el dominio del problema de manera ordenada. En vez de tener un solo archivo con todo el código, dividimos el sistema en partes: cada clase representa una entidad (o responsabilidad) y los objetos son los que “viven” en la ejecución del programa.

En la vida real En POO Ejemplo
Tipo de entidad Clase “Producto”, “Cliente”, “Reserva”
Entidad específica Objeto Producto #1, Cliente “María”, Reserva #204
Crear una entidad Instanciar new Producto(...)
Conjunto de entidades Colección (más adelante) Lista de productos, lista de reservas

1.2. Identidad, estado y comportamiento (sin entrar en detalle aún)

Todo objeto tiene tres ideas importantes. En esta semana nos enfocaremos en las dos primeras de forma conceptual, porque el detalle de estado (atributos), comportamiento (métodos) y protección (encapsulación) se desarrolla a profundidad en la semana 3.

Identidad

  • Un objeto es “ese objeto”, único en memoria.
  • Dos objetos pueden parecer iguales, pero ser diferentes instancias.
  • En Java, una variable puede apuntar (referenciar) a un objeto.

Estado y comportamiento

  • Estado: datos que describen al objeto (veremos en semana 3).
  • Comportamiento: acciones del objeto (veremos en semana 3).
  • Hoy nos centraremos en cómo “existir” como objeto y cómo se crea.

2. Clase vs Objeto (con ejemplos claros en Java)

Para comprenderlo bien, es clave no mezclar estos conceptos: la clase es código, el objeto es memoria. Una clase por sí sola no “hace nada” hasta que se crean objetos a partir de ella.

2.1. Una clase mínima en Java

Una clase puede existir incluso sin atributos y métodos, aunque en la práctica casi siempre tendrá. En esta semana lo importante es que veas la estructura básica: 

package com.corhuila.poo.semana2;

public class Libro {
    // (En semana 3 veremos atributos y métodos en detalle)
}

2.2. Crear objetos (instancias) con new

Para que exista un objeto, Java necesita reservar memoria. Eso ocurre cuando usas: new NombreDeLaClase(). 

package com.corhuila.poo.semana2;

public class App {
    public static void main(String[] args) {
        Libro l1 = new Libro(); // objeto 1
        Libro l2 = new Libro(); // objeto 2

        System.out.println(l1);
        System.out.println(l2);
    }
}

Aunque l1 y l2 son del mismo tipo (Libro), son objetos diferentes. En consola verás algo parecido a com.corhuila.poo.semana2.Libro@1a2b3c, que representa la referencia del objeto (una forma de identificarlo en memoria).

2.3. Variables de referencia (muy importante para evitar errores)

En Java, cuando haces Libro l1 = new Libro();, la variable l1 NO “contiene” el objeto como tal. Contiene una referencia (una dirección/identificador) que apunta al objeto.

Referencia: variable que apunta a un objeto en memoria.
Si dos referencias apuntan al mismo objeto, entonces están “viendo” el mismo objeto.

Ejemplo: dos referencias al mismo objeto

package com.corhuila.poo.semana2;

public class App {
    public static void main(String[] args) {
        Libro l1 = new Libro();
        Libro l2 = l1; // l2 apunta al MISMO objeto que l1

        System.out.println(l1);
        System.out.println(l2); // imprimen algo muy similar (misma referencia)
    }
}

Este concepto será fundamental cuando trabajes con listas de objetos, actualizaciones y comparaciones. Muchos errores de principiantes vienen de no entender que una variable puede apuntar al mismo objeto que otra.

2.4. El valor null: referencia sin objeto

En Java, una referencia puede existir sin apuntar a ningún objeto. En ese caso vale null. Si intentas usar una referencia null como si fuera un objeto, ocurre el famoso error: NullPointerException. 

package com.corhuila.poo.semana2;

public class App {
    public static void main(String[] args) {
        Libro libro = null;

        // Cuidado: libro no apunta a ningún objeto.
        // En semana 3 verás cómo evitar estos errores con validaciones y buenas prácticas.
        System.out.println(libro);
    }
}
Buenas prácticas desde ya
  • Evita dejar referencias en null si puedes inicializarlas.
  • Antes de usar un objeto, asegúrate de que exista (no sea null).
  • Si algo “no existe”, decide si null es adecuado o si necesitas otro enfoque (más adelante).
Errores típicos
  • Creer que l2 = l1 crea un objeto nuevo (no lo crea).
  • Confundir “tipo” (Libro) con “instancia” (new Libro()).
  • No entender por qué dos variables cambian “juntas” (porque apuntan al mismo objeto).

3. Construcción en Java: organizar clases, paquetes y ejecución

En proyectos reales, no se trabaja con una sola clase. Incluso un programa pequeño debería estar organizado: una clase principal (App) y clases de modelo (por ejemplo, Libro, Autor, etc.). Esto se logra mediante paquetes.

3.1. Paquetes (packages): qué son y por qué se usan

Un paquete es una forma de agrupar clases relacionadas. Además de ordenar, ayuda a evitar conflictos cuando dos clases tienen el mismo nombre en diferentes partes del proyecto.

Paquete: un “contenedor lógico” de clases.
Ejemplo: com.corhuila.poo.semana2 agrupa todo lo que trabajas en esta semana.

Estructura típica:

src/
  com/
    corhuila/
      poo/
        semana2/
          App.java
          Libro.java

3.2. Flujo típico para crear objetos en un programa

En la mayoría de programas (a nivel básico), el flujo inicia en main: se crean objetos y luego se ejecuta una lógica inicial sobre ellos. Por ahora nos interesa dominar el patrón:

Paso Qué haces Ejemplo
1 Definir una clase (molde) public class Libro { }
2 Crear objetos (instancias) Libro l1 = new Libro();
3 Guardar referencias Libro l2 = l1;
4 Verificar que entiendes identidad Objetos distintos vs referencias al mismo objeto

3.3. Recomendaciones para esta semana (sin adelantarnos a encapsulación)

Todavía no vamos a profundizar en atributos y métodos, pero sí puedes practicar con clases “vacías” y con clases que tengan lo mínimo para poder crear instancias correctamente. En la semana 3 aprenderás a diseñar el interior de la clase con atributos privados, métodos públicos y validaciones.

Enfoque correcto de Semana 2
  • Entender diferencia clase vs objeto.
  • Crear múltiples instancias con new.
  • Entender referencias: copiar referencia no crea objeto.
  • Identificar qué significa null y el riesgo del NullPointerException.
Lo que NO vamos a profundizar aún
  • Diseño completo de atributos (estado) con reglas.
  • Métodos con lógica de negocio y validaciones completas.
  • Encapsulación (private, getters/setters) en profundidad.
  • Herencia, polimorfismo, interfaces (vienen después).

Actividad práctica de la semana (no obligatoria)

1. Contexto

Vas a crear un mini-proyecto en Java para practicar la diferencia entre clase y objeto y dominar el uso de new, referencias y null. En esta actividad NO se evaluará aún encapsulación ni diseño avanzado de métodos (eso se trabaja en la semana 3).

  1. Crea un proyecto con paquete com.corhuila.poo.semana2.
  2. Crea la clase Producto (puede estar vacía o con una estructura mínima).
  3. En App, crea varios objetos y prueba referencias.

2. Requerimientos del mini-ejercicio “Catálogo simple”

Implementa dos clases: Producto y App. El foco es la creación de objetos y el manejo de referencias.

Requisito Qué debes hacer Qué se busca evidenciar
R1. Clase Producto Crear la clase Producto (sin lógica compleja). Definir una clase como “molde”.
R2. Crear instancias Crear 3 objetos Producto con new. Diferenciar clase vs objeto.
R3. Probar referencias Asignar Producto p2 = p1; y mostrar que no es un objeto nuevo. Entender referencias y misma identidad.
R4. Probar null Crear una referencia Producto pNull = null; e imprimirla. Entender referencia sin objeto (sin provocar errores).

3. Entregables

Entregable Qué incluye Formato sugerido
Código fuente Proyecto con Producto.java y App.java. .zip o repositorio
Salida en consola Impresión de referencias para evidenciar objetos distintos o iguales. Texto o captura
Breve explicación 5–8 líneas explicando: ¿qué pasó con p2 = p1? ¿qué significa null? Texto / Markdown

Criterios de evaluación (referencia)

Se observará: (1) que el proyecto compile y ejecute, (2) uso correcto de new, (3) comprensión de referencia vs objeto, (4) claridad en la explicación final. No se calificará aún encapsulación ni diseño completo de atributos/métodos.
