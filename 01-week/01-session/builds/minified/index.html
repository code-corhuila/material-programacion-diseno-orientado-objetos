Semana 1: Introducción a la POO y entorno de desarrollo en Java

En esta primera semana construiremos la base del curso: comprenderás qué es la Programación Orientada a Objetos (POO), por qué es el paradigma dominante en la industria y cómo se traduce en Java mediante clases, objetos, atributos y métodos. Además, dejarás listo tu entorno de desarrollo para programar de forma correcta y profesional.

1. Fundamentos de la Programación Orientada a Objetos (POO)

La POO es un paradigma de programación que propone construir software modelando el mundo real (o el dominio del problema) mediante objetos que tienen estado (datos) y comportamiento (acciones). En lugar de pensar el programa como una lista de instrucciones sueltas, se organiza como un conjunto de entidades que colaboran entre sí.

Objeto: una entidad con identidad propia que combina datos (estado) y métodos (comportamiento).
Clase: un “molde” o plantilla que define cómo serán los objetos: qué datos tendrán y qué acciones podrán realizar.

1.1. ¿Por qué existe la POO?

A medida que un sistema crece, la complejidad aumenta: más reglas, más módulos, más cambios. La POO aparece como una forma de organizar el código para que sea más mantenible, reutilizable y comprensible. En sistemas reales, lo más costoso no es “programar una vez”, sino mantener y evolucionar el software durante años.

Necesidad en software Cómo ayuda la POO Ejemplo práctico
Organizar la complejidad Divide el sistema en objetos con responsabilidades claras. En un sistema de compras: Producto, Carrito, Pago.
Reutilizar código Permite crear clases reutilizables y extender comportamientos. Un Vehículo base y sus variantes: Carro, Moto.
Reducir errores Encapsula datos para protegerlos y controlar cambios mediante métodos. No permitir saldo negativo en una CuentaBancaria.
Facilitar cambios Modifica una clase sin romper todo el sistema (si el diseño es correcto). Cambiar la lógica de descuento sin tocar otras partes.

1.2. POO vs. programación estructurada (visión clara)

No significa que la programación estructurada sea “mala”: de hecho, la POO se apoya en ella (condicionales, ciclos, funciones). La diferencia está en cómo organizamos el sistema. Mira la comparación:

Aspecto Estructurada (procedimental) Orientada a Objetos
Organización Funciones/procedimientos que procesan datos. Objetos que encapsulan datos y métodos.
Enfoque “¿Qué pasos debo ejecutar?” “¿Qué entidades existen y qué hacen?”
Datos Frecuentemente globales o pasados entre funciones. Protegidos dentro del objeto (encapsulación).
Escalabilidad Puede complicarse en sistemas grandes por acoplamiento. Mejor organización para sistemas con muchas reglas.

1.3. Los 4 pilares de la POO (vista general, con sentido)

Durante la materia profundizaremos en estos pilares. Por ahora, necesitas comprender qué significan y por qué importan. No se trata de “memorizar definiciones”, sino de entender cómo mejoran tu diseño.

Encapsulación

  • Proteger el estado interno del objeto.
  • Acceder/modificar datos solo mediante métodos controlados.
  • Evita cambios inválidos (reglas del negocio).

Idea clave: “No toques mis datos directamente; usa mis métodos”.

Abstracción

  • Modelar lo esencial y ocultar detalles innecesarios.
  • Definir qué hace una entidad, no cómo lo hace internamente.
  • Ayuda a pensar en “responsabilidades”.

Idea clave: trabajar con conceptos claros del dominio.

Herencia

  • Crear clases nuevas basadas en clases existentes.
  • Reutiliza atributos y métodos comunes.
  • Permite especialización (general → específico).

Idea clave: “Un Carro ES un Vehículo” (relación “es-un”).

Polimorfismo

  • Una misma operación puede comportarse distinto según el objeto.
  • Permite escribir código general que funciona con múltiples tipos.
  • Es clave para diseño flexible y extensible.

Idea clave: “Mismo método, diferentes comportamientos”.

1.4. Primer modelo mental: “clase” y “objeto” en un ejemplo real

Supongamos que queremos modelar una CuentaBancaria. En la vida real, una cuenta tiene datos (saldo, número) y acciones (depositar, retirar). En POO, eso se convierte en una clase con atributos y métodos.

Clase CuentaBancaria: define qué es una cuenta en nuestro sistema.
Objeto cuentaDeAna: es una cuenta específica con un saldo específico.

Ejemplo (Java):

public class CuentaBancaria {
    private String numero;
    private double saldo;

    public CuentaBancaria(String numero, double saldoInicial) {
        this.numero = numero;
        this.saldo = saldoInicial;
    }

    public void depositar(double monto) {
        if (monto <= 0) {
            System.out.println("El monto a depositar debe ser mayor a 0.");
            return;
        }
        saldo += monto;
    }

    public void retirar(double monto) {
        if (monto <= 0) {
            System.out.println("El monto a retirar debe ser mayor a 0.");
            return;
        }
        if (monto > saldo) {
            System.out.println("Fondos insuficientes.");
            return;
        }
        saldo -= monto;
    }

    public double getSaldo() {
        return saldo;
    }
}

Observa lo importante: el saldo está encapsulado (private). Nadie debería cambiarlo haciendo saldo = -999 desde afuera. Solo se modifica mediante depositar y retirar, donde aplicamos reglas de negocio.

Uso desde un programa principal:

public class App {
    public static void main(String[] args) {
        CuentaBancaria cuenta = new CuentaBancaria("ACC-001", 500);

        cuenta.depositar(200);
        cuenta.retirar(100);
        cuenta.retirar(1000); // fondos insuficientes

        System.out.println("Saldo final: " + cuenta.getSaldo());
    }
}

2. Entorno de desarrollo en Java (JDK, JVM, IDE y ejecución)

Para trabajar correctamente en Java necesitas entender qué se instala y qué hace cada componente. Muchas confusiones comunes vienen de no diferenciar JDK, JRE y JVM.

JVM (Java Virtual Machine): ejecuta el bytecode (.class) en cualquier sistema operativo.
JRE (Java Runtime Environment): JVM + librerías necesarias para ejecutar programas Java.
JDK (Java Development Kit): JRE + herramientas de desarrollo (compilador javac, depurador, etc.).
Conclusión: para programar necesitas JDK.

2.1. ¿Cómo se ejecuta un programa Java realmente?

Java no se ejecuta directamente como “código fuente”. El proceso típico es:

Etapa Qué ocurre Archivo resultante
1) Escribir código Creas clases en archivos fuente. .java
2) Compilar El compilador traduce .java a bytecode. .class
3) Ejecutar La JVM interpreta/ejecuta el bytecode. Programa corriendo

Ejemplo por consola (sin IDE):

javac App.java
java App

En un IDE (IntelliJ/Eclipse/VS Code), este proceso ocurre “por detrás”, pero sigue siendo el mismo. Entenderlo te ayuda a depurar errores de configuración y a comprender cómo funciona el lenguaje.

2.2. Instalación y verificación del JDK (pasos prácticos)

Instala un JDK reciente (preferiblemente una versión LTS). Luego verifica en la terminal: 

java -version
javac -version

Si estos comandos no funcionan, normalmente significa que: (1) el JDK no está instalado, o (2) la variable PATH no apunta a la carpeta bin del JDK.

2.3. IDE recomendado y por qué

Puedes programar Java con varios entornos, pero para aprender POO con buenas prácticas (proyectos, paquetes, depuración, refactorización), un IDE completo acelera el aprendizaje.

Herramienta Ventajas Recomendación de uso
IntelliJ IDEA Excelente para Java, refactorizaciones y debugging, muy didáctico. Ideal para el curso (Community Edition es suficiente).
Eclipse Robusto, clásico en academia e industria, muchos plugins. Buena alternativa si ya lo conoces.
VS Code Ligero, requiere extensiones, funciona bien con proyectos simples. Útil si tu equipo es limitado, pero exige más configuración.

2.4. Estructura mínima de un proyecto Java (conceptos que debes dominar)

Incluso en proyectos pequeños, Java promueve organización por paquetes (packages). Un paquete agrupa clases relacionadas y evita conflictos de nombres.

Convención de paquetes: se recomienda usar formato tipo dominio invertido.
Ejemplo: com.corhuila.poo.semana1

Estructura típica:

mi-proyecto/
  src/
    com/
      corhuila/
        poo/
          semana1/
            App.java
            CuentaBancaria.java

Esta estructura te prepara para proyectos más grandes donde tendrás múltiples clases y responsabilidades.

3. Primeros pasos en Java orientado a objetos (con buenas prácticas)

En esta sección unimos las ideas: escribiremos un programa simple, pero organizado como se trabaja en POO: clases separadas, responsabilidades claras y uso correcto de atributos/métodos.

3.1. La clase principal y el método main

En Java, el punto de entrada de un programa de consola es el método: public static void main(String[] args). Desde ahí puedes crear objetos y ejecutar la lógica inicial. 

package com.corhuila.poo.semana1;

public class App {
    public static void main(String[] args) {
        System.out.println("Bienvenido a POO en Java");
    }
}

3.2. Reglas básicas de diseño (que aplicaremos todo el curso)

Reglas de oro para empezar
  • Una clase = una responsabilidad principal (no mezcles todo en App).
  • Atributos privados y acceso mediante métodos.
  • Nombres claros (CuentaBancaria, depositar, retirar).
  • Validar datos en el lugar correcto (en el método, no “afuera”).
Errores comunes del inicio
  • Hacer todo en main (se vuelve inmantenible rápido).
  • Usar variables públicas para todo (pierdes control del estado).
  • No modelar el dominio (terminas con “código spaghetti”).
  • Copiar y pegar lógica repetida en vez de encapsularla.

3.3. Ejemplo guiado: clase Mascota (estado + comportamiento)

Vamos a modelar una Mascota. Este ejemplo es intencionalmente simple, porque el objetivo es dominar la idea de “objeto con estado y acciones”. 

package com.corhuila.poo.semana1;

public class Mascota {
    private String nombre;
    private String especie;
    private int energia; // 0 a 100

    public Mascota(String nombre, String especie) {
        this.nombre = nombre;
        this.especie = especie;
        this.energia = 50; // energía inicial por defecto
    }

    public void comer(int cantidad) {
        if (cantidad <= 0) {
            System.out.println("La cantidad debe ser mayor a 0.");
            return;
        }
        energia = Math.min(100, energia + cantidad);
        System.out.println(nombre + " comió. Energía: " + energia);
    }

    public void jugar(int minutos) {
        if (minutos <= 0) {
            System.out.println("Los minutos deben ser mayor a 0.");
            return;
        }
        energia = Math.max(0, energia - minutos);
        System.out.println(nombre + " jugó. Energía: " + energia);
    }

    public String resumen() {
        return "Mascota{nombre='" + nombre + "', especie='" + especie + "', energia=" + energia + "}";
    }
}

Uso desde App:

package com.corhuila.poo.semana1;

public class App {
    public static void main(String[] args) {
        Mascota m = new Mascota("Luna", "Perro");

        System.out.println(m.resumen());
        m.comer(30);
        m.jugar(20);
        System.out.println(m.resumen());
    }
}

Nota cómo la energía se mantiene consistente gracias a: encapsulación (private) + métodos que aplican reglas (límites 0–100). Esto es POO: el objeto cuida su estado.

3.4. Qué debes dominar al finalizar esta semana

Competencia Qué significa Cómo lo evidencias
Diferenciar clase y objeto Entender plantilla vs instancia real. Creas 2 objetos de la misma clase con estados distintos.
Encapsular atributos Usar private y controlar cambios con métodos. No modificas atributos desde afuera; usas métodos.
Configurar el entorno JDK instalado y proyecto ejecutando sin errores. Evidencias java -version y ejecución en IDE.
Separar responsabilidades No concentrar todo en main. Tienes clases separadas para el modelo y la ejecución.

Actividad práctica de la semana (no obligatoria)

1. Contexto

Vas a preparar tu entorno de trabajo y a construir tu primer mini-programa orientado a objetos en Java. El objetivo no es hacer “algo enorme”, sino demostrar que sabes modelar una entidad como clase, crear objetos y ejecutar métodos que controlan el estado.

  1. Instala y verifica el JDK (comandos java -version y javac -version).
  2. Crea un proyecto en tu IDE con un paquete (ej.: com.corhuila.poo.semana1).
  3. Implementa el mini-programa “Gestor de Tareas” (clases + ejecución).

2. Requerimientos del mini-programa “Gestor de Tareas”

Vas a construir dos clases: Tarea y App.

Clase Qué debe incluir Reglas
Tarea Atributos: titulo, descripcion, completada.
Métodos: marcarComoHecha(), marcarComoPendiente(), resumen(). 		Los atributos deben ser private. El resumen debe mostrar el estado.
App Crear al menos 3 objetos Tarea y ejecutar métodos sobre ellos. Debe imprimirse en consola el estado antes y después de cambios.

3. Entregables

Entregable Qué incluye Formato sugerido
Evidencia de entorno Captura o copia del resultado de java -version y javac -version. Imagen / PDF
Código fuente Proyecto con clases Tarea y App ejecutables. Carpeta comprimida (.zip) o repositorio
Salida en consola Pruebas: imprimir estados antes/después de completar tareas. Texto / Captura

Criterios de evaluación (referencia)

Esta actividad no es obligatoria, pero si la realizas te ayudará a fortalecer la base del curso. Se revisará principalmente: (1) entorno configurado, (2) uso correcto de clases/objetos, (3) encapsulación (private), (4) claridad del código y nombres, (5) evidencias de ejecución.
